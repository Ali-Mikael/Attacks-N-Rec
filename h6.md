## B) Report on your findings     
**Recommended tools:**
- `wifi_db`
  - <https://github.com/r4ulcl/wifi_db>
  - **README:** "*Script to parse Aircrack-ng captures into a SQLite database and extract useful information like handshakes (in 22000 hashcat format), MGT identities, interesting relations between APs, clients and it's Probes, WPS information and a global view of all the APs seen.*"
- `aircrack-ng`
  - <https://github.com/aircrack-ng/aircrack-ng?tab=readme-ov-file>
  - **README:** "*Aircrack-ng is a complete suite of tools to assess WiFi network security.*"
    - Monitoring, Attacking, Testing & Cracking
- `airgeddon`
  - <https://github.com/v1s1t0r1sh3r3/airgeddon>
  - Out of a long list of features, a few..:
    - WPS scanning, Decloak of hidden ESSID networks, Cleaning and optimizing Handshake captured files.
    - WPA3 online dictionary attack, Evil Twin attacks, Simple integrated sniffing (Hostapd + DHCP + DoS + Ettercap), DoS over wireless networks
- `eaphammer`
  - <https://github.com/s0lst1c3/eaphammer>
  - **README:** "*EAPHammer is a toolkit for performing targeted evil twin attacks against WPA2-Enterprise networks.*"
- `hashcat`
  - <https://hashcat.net/hashcat/>
  - **README:** *"hashcat is the world's fastest and most advanced password recovery utility."*

I fired up the VM and started exploring with some help from the user friendly *walkthrough* provided: <https://r4ulcl.com/posts/walkthrough-wifichallenge-lab-2.0/>       

Everything started with the `airmon-ng` command. 
**Q:** But what is it?      
**A:** Straight from the man pages: **"*airmon-ng* This script can be used to enable monitor mode on wireless interfaces."**     
I ran some commands and played detective.

I wanted to know more about the `wifi-corp`, so I took up the channel (44) and ran the command with the channel parameter
```
sudo airodump-ng wlan0mon -w ~/wifi/channel44 --manufacturer --wps -c44
```
- The `-w ~/wifi/channel44` saves capture in the `wifi directory` under name `channel44`.
